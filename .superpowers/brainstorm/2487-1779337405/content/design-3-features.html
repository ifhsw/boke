设计第3部分：交互细节 & 功能清单

编辑器核心功能和交互行为

⌨️ 输入交互

交互行为
Markdown 快捷输入输入 # 空格 → H1，## → H2，**文字** → 加粗，`代码` → 行内代码，> → 引用块 等。TipTap 内置支持。
斜杠命令 /行首输入 / 弹出菜单：H1/H2/H3、图片、代码块、表格、Callout、数学公式、分割线、列表、任务列表、引用
浮动菜单选中文字后浮现 Bubble Menu：加粗、斜体、删除线、链接、行内代码
键盘快捷键Ctrl+B 加粗、Ctrl+I 斜体、Ctrl+K 链接、Ctrl+Z 撤销、Ctrl+Shift+Z 重做、Ctrl+S 手动保存

🖼 图片处理

场景实现
拖拽上传拖拽图片到编辑区 → 自动上传到 /api/upload → 插入图片节点。上传中显示占位 loading。
粘贴截图Ctrl+V 粘贴剪贴板图片 → 自动上传 → 插入。复用 handlePaste 钩子。
工具栏按钮点击工具栏图片按钮 → 打开文件选择器 → 上传 → 插入。
封面图上传独立的上传区域（顶部虚线框），与编辑器共享同一个 /api/upload 接口。上传后存储 URL 到 coverImage 字段。
图片节点交互点击图片选中 → 可拖拽调整大小 / 设置标题 (caption) / 删除。

💾 自动保存

机制说明
本地草稿 (localStorage)每次内容变化后 3 秒防抖 → 序列化编辑器 JSON 到 localStorage。刷新页面/意外关闭后可恢复。
服务端草稿每 30 秒自动保存到服务端 (status=DRAFT)，或手动 Ctrl+S 立即保存。
状态指示灯顶部栏绿色圆点 + "已保存" / "保存中..." / "未保存" 文字。
草稿恢复提示新建文章时检测 localStorage 是否有未发布的草稿 → 弹窗 "检测到未保存的草稿，是否恢复？"

🧩 自定义 TipTap 扩展 (替代原 MDX 组件)

扩展说明渲染 (NodeView)
Callout 4 种类型：Info (蓝)、Warn (黄)、Tip (绿)、Note (灰)。通过 / 菜单或工具栏插入。 带颜色左边框 + 图标 + 背景色的卡片
CodeBlock (Shiki) 代码块 + 语言选择器 + Shiki 高亮。编辑时显示语言标签。 macOS 三点 + 文件名栏 + 深色背景代码区
ImageCaption 图片 + 描述文字。替代原 MDX ImageCaption。 圆角图片 + 下方灰色居中描述
Math (KaTeX) 行内公式 + 块级公式。通过 / 菜单或 $$ 快捷输入。 KaTeX 渲染的数学公式

📋 标签输入组件

替换当前逗号分隔的 input。独立的多标签输入组件：

  • 输入文字后按 回车 或 逗号 确认一个标签
  • 每个标签显示为 chip/badge，带 × 删除按钮
  • 自动补全已有标签（从数据库查询）
  • 点击已有标签直接添加

⏰ 定时发布

  • 设置 scheduledAt 后，状态保持 DRAFT
  • 非管理员投稿强制 DRAFT + 审核，不直接支持定时发布
  • 前端倒计时显示 "将于 2026-05-22 10:00 发布"
  • 管理员可直接设定 PUBLISHED + 未来时间 = 预发布状态
