三种编辑器实现方案

约束：必须保持 MDX 兼容，不破坏已有文章内容

A

Milkdown — WYSIWYG Markdown 编辑器 (推荐)

基于 ProseMirror，输出纯 Markdown 字符串，与现有 MDX 管线无缝对接。

Pros

  • 输出 Markdown，零迁移成本，MDX 渲染不动
  • 内置工具栏、slash 命令、拖拽粘贴图片
  • 插件生态丰富 (toolbar, clipboard, emoji, prism, math)
  • 所见即所得 + Markdown 快捷输入双模式
  • 包体积适中 (~150KB gzipped)

Cons

  • ProseMirror 有一定学习曲线
  • 自定义 MDX 组件 (Callout/Info/Warn) 需要额外适配
  • 中文输入法兼容需要测试
B

TipTap — 富文本编辑器

同样基于 ProseMirror，生态最大，但默认输出 JSON/HTML，需转换层输出 Markdown。

Pros

  • 最大社区，文档最完善
  • 扩展极丰富 (table, image, code-block, collaboration)
  • React 集成成熟 (@tiptap/react)
  • 中文支持经过大量验证

Cons

  • 默认存储格式是 JSON，需引入 Markdown 序列化插件
  • Markdown 序列化可能丢失高级结构 (Callout, Info/Warn 组件)
  • 包体积更大 (~200KB+ gzipped)
  • 维护两套渲染逻辑风险 (MDX 渲染 + TipTap 渲染)
C

增强型 Textarea — 最小改动

保留现有 textarea，叠加工具栏 + 实时预览 + 图片拖拽，最低风险。

Pros

  • 几乎不改现有架构
  • 无学习曲线，Markdown 纯文本永远可控
  • 最轻量，无额外依赖
  • MDX 自定义组件语法直接手写

Cons

  • 不是真正的所见即所得
  • 预览和编辑分离 (两栏或切换)
  • 长文滚动时编辑/预览位置同步难
  • 体验提升有限

推荐理由：Milkdown (A)

对于这个项目，内容格式兼容性比功能丰富度更重要。已有文章 + MDX 自定义组件 (Callout, CodeBlock, ImageCaption) 是核心资产。Milkdown 输出 Markdown 字符串，MDX 管线完全不动；TipTap 的 JSON 存储会引入不必要的转换风险。增强型 textarea 改善有限，不值得专门做一个迭代。
